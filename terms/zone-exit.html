Zone Exit

Geofencing & Zones

An event generated when a tracked tag leaves a defined geofence zone. Triggers alerts, logs events, or initiates automated actions. Use cases include security (unauthorized asset removal), process control (verifying completion), and safety (tracking exposure time).

Zone exit in industrial RTLS is a zone detection event that triggers when a tracked asset or person transitions from inside to outside a defined geographical zone boundary.

Technical considerations for reliable zone exit detection mirror entry detection: positioning accuracy determines minimum zone size, update rate affects exit detection latency, boundary buffers prevent false exits from positioning uncertainty, and exit confirmation logic requires sustained absence vs. momentary detection outside boundaries preventing false triggers from positioning variability. Zone exit analytics provide process insights: exit frequency measuring how often zones are left (activity levels by area), exit timing patterns revealing when exits typically occur (shift change indicators, process cycle rhythms), average dwell time (time from entry to exit) indicating process efficiency, exit-entry matching verifying workflow continuity (materials exiting zone A should enter downstream zone B), and exception rates tracking unexpected exits (premature departures, missing exits suggesting assets remained behind). Best practices for zone exit configuration include: exit confirmation thresholds requiring multiple consecutive position updates outside boundary before confirming exit, temporal filtering requiring tags remain outside zone minimum duration (5-30 seconds) before confirming permanent exit, hysteresis using different boundary thresholds for exits vs. entries (exit boundary slightly inside entry boundary preventing rapid toggling), and alert configuration distinguishing critical exits (unauthorized removals) from routine exits (normal workflow progression). Pairing zone entry and exit events creates comprehensive zone lifecycle tracking: entry marks beginning, multiple position updates during dwell confirm presence, exit marks conclusion, and analytics on complete cycle reveal process performance (dwell time, throughput, compliance).
