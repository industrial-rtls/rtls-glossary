Gateway

Hardware

A network device bridging between RTLS infrastructure components and central systems. Collects data from multiple anchors or tags and forwards it to servers or cloud platforms. Functions include data aggregation, protocol translation, data preprocessing, and network management. Enables scalable architectures and simplifies network topology.

Network device bridging wireless RTLS tags/sensors and wired infrastructure, aggregating data from multiple tags and forwarding to central systems. In BLE/Wi-Fi RTLS deployments, gateways perform: signal reception from battery-powered tags (typically 20-50 meter radius), data aggregation (collecting signals from 50-500 tags), local processing (RSSI measurement, timestamp assignment, data filtering), and network forwarding (transmitting aggregated data to position engines via Ethernet/Wi-Fi). Gateway types: (1) Dedicated RTLS gateways - purpose-built devices optimized for RTLS performance, supporting specialized positioning features (AoA antenna arrays, precise timestamping). Cost $100-500 per gateway. (2) Standard IoT gateways - general-purpose BLE/Wi-Fi gateways supporting RTLS among other applications. Cost $50-200 per gateway. (3) Integrated gateways - RTLS functionality embedded in existing devices (access points, edge servers). Minimal incremental hardware cost. Gateway specifications include: coverage area (20-50 meters typical for BLE, 30-100 meters for Wi-Fi), tag capacity (simultaneous tags), update rate (1-10 Hz), network interface (10/100/1000 Mbps Ethernet, Wi-Fi), power (PoE most common for deployment simplicity), environmental rating (IP65-67 for industrial environments). Gateway placement critical: similar principles as anchor placement but typically lower density (1 gateway per 500-2000 m²) due to longer range and lower accuracy requirements. Gateway deployment simpler than UWB anchor deployment - no precise positioning required (gateway locations need not be surveyed), no synchronization needed, and simplified commissioning (connect to network, configure, operational).
